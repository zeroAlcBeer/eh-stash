
  老 TF-IDF 那版只是"能工作"，但有几个结构性硬伤现在都被掉根了：

  1. 阈值有了稳定语义，可以代际比较

  老：score = Σ匹配tag权重，无上界。20 这个阈值是经验数字，每次 RebuildPreferenceTags 之后 IDF 基数会变、tf 分布会变 → 阈值会"漂移"。你今天调好的
  20，下周可能要调成 25 才合适。

  新：similarity ∈ [0, 1]，永远是单位向量夹角的余弦。改 type_weight、加新画廊、加新收藏，分数尺度不变。6 个月后的 0.08 和今天的 0.08 是同一回事。

  2. 几何归一化消掉了 tag 数量偏置

  老：30 个 tag 的画廊 vs 10 个 tag 的画廊，前者命中概率更高 + 累加分天然更高。系统对"tag 堆得多"的画廊有结构性偏好——而这恰好和 eh 上"高质量原创作品
   tag 反而少"的现实相反。

  新：L2 归一化让每个画廊的"能量"等于 1。dual sparse 30 个 tag 的画廊和 10 个 tag 的画廊比的是方向，不是长度。系统不再被 tag 数量绑架。

  3. 描述性 namespace 从二元决策变成连续权重

  老：硬编码只看 artist/group/character/parody。female/male/language/other 等等完全不参与召回——这意味着一个画廊就算和你品味题材高度吻合，但 artist
  没碰过，就完全推不出来。

  新：所有 namespace 都参与，由 type_weight 控制强度（artist 1.7 → female 0.6 → language 0.2）。召回宽度大幅扩大，同时主信号仍然是
  artist/character——因为 type_weight 进入点积是平方效应，相对差距比看上去的大得多（1.7² / 0.6² ≈ 8×）。

  4. IDF 隐含被平方了，稀有信号被放大

  老 TF-IDF：score += tf_weight × idf，IDF 线性出现一次。

  新 cosine：score = Σ (idf · tw / ‖G‖) × (idf · tw · (1+ln(tf)) / ‖P‖)——IDF 在两侧都出现，相当于平方。一个 df=3 的小众 artist
  命中，对最终得分的贡献是 df=2000 的 female 描述 tag 的几十倍，而老系统只是十倍量级。"小众但准确"的信号更容易冒头。

  5. 计算和源数据架构分离

  老：eh_galleries 啥都没有，但 preference_tags + recommended_cache 各自一套逻辑、各自一套 rebuild 时机，缺一个分数就不对。

  新：
  eh_galleries    ← 只放从 EH 抓回的源数据
  tag_vocabulary  ← 词表（系统给定的语义空间）
  user_profile    ← 用户偏好向量（你的口味）
  recommended_cache
    ├ tag_embedding ← 画廊在空间里的位置
    └ similarity    ← 你和画廊的距离
  每张表职责单一，可独立 rebuild。配 worker 的 partial / full recompute，永远收敛到正确状态。

  6. 预计算 + 索引让请求路径不算 cosine

  老 reverse-scorer 是"批量重算分数表"，思路对了，但 SQL 还是把 tag-match 当 string 比较（g.tags @> jsonb_...）。

  新方案把这一步彻底从请求路径拿走：similarity 是 REAL 列 + partial b-tree。distribution 是 width_bucket 单列扫，For You 是 WHERE similarity >= ?
  ORDER BY similarity DESC 走索引范围扫。

  distribution: 6,651ms → 71ms  (-99%)
  for_you:      3,070ms → 65ms  (-98%)

  7. profile 是单一稠密向量，不是表

  老 preference_tags 是一张表，算分数要 JOIN 几万条偏好行。新 profile 是 1 个 65536 维稀疏向量，一次点积出结果。这是为啥能享受 pgvector ANN
  的潜在收益（虽然当前规模不需要 ANN）。

  8. profile 不再依赖 embedding 完成度

  最隐蔽但最重要的一条。老系统 reverse-scorer 必须等 preference_tags 写好才能跑分。新 RebuildUserProfile 走纯 TF-IDF SQL 公式，只读 user_favorites +
   eh_galleries.tags + tag_vocabulary——favorites 同步完那一刻 profile 就可用，不用等任何 embedding
  worker。这让"刚收藏一个新作者立刻看到推荐变化"这个体验成为现实。

  ---
  一句话总结

  老版本是"把 TF-IDF 思想拼凑成一个能用的推荐系统"——能跑，但每个组件都凑合（阈值漂移、长度偏置、namespace 二元、混表）。

  新版本是"用 cosine 几何把 TF-IDF 思想原本想表达的东西干净表达出来"——同样的核心数学（(1+ln(tf))·idf
  还在那儿），但放到向量空间里之后，归一化、可比阈值、命名空间分级、架构分离、预计算可索引这些全部自然涌现。

  唯一保留老版本"原汁原味"的，是 (1+ln(tf)) 这个亚线性 TF 压制——这是老版本做对的核心 insight，被精确移植到了 profile 那一步。
